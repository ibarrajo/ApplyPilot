# Resume Guidelines (Jobscan Methodology)

## tl;dr

- **Target an 80% Jobscan Match Rate** (75% minimum floor) against the specific job description. Above ~85% risks keyword stuffing and hurts human readability.
- **Single-column, plain text, standard section headers, no graphics/tables/text boxes/headers-footers.** ATS parsers read top-to-bottom, left-to-right as a linear stream — anything that violates that gets scrambled or dropped.
- **Mirror the job title and hard skills verbatim from the posting.** Candidates with the exact job title on their resume get **10.6x more interviews**. Hard skills beat soft skills — 76.4% of recruiters filter by skill, 88% of hiring managers prioritize hard skills.

---

## 1. Match Rate Targets

**Aim for 80% match.** Jobscan calls this "the sweet spot between optimizing your resume without overstuffing it with keywords." A 75% match is still competitive but not as optimized.

Source: https://www.jobscan.co/blog/what-jobscan-match-rate-should-i-aim-for/

**Do not exceed ~85% by forcing keywords.** Jobscan explicitly warns against:

1. Using keywords inaccurately or falsely to inflate scores
2. Claiming unproven skills
3. Creating awkward, robotic-sounding text through excessive repetition
4. Hiding keywords in white font (unethical, increasingly detected)
5. Overusing specific terms to the point it destroys readability

Why this ceiling exists: humans read the resume after the ATS surfaces it. An over-tuned resume reads like spam.

**Context for the 80% number:** Jobscan reports **70% of resumes get rejected by the ATS before humans see them**. Optimization is not optional — it's table stakes.

## 2. File Format (PDF vs DOCX)

**Default to PDF. Keep DOCX as a fallback.** Jobscan's explicit recommendation:

> "We recommend converting your Word resume into PDF when applying for a job, unless the job posting specifically requests a .docx file."

Source: https://www.jobscan.co/blog/resume-pdf-vs-word/

### PDF Advantages
- Formatting preserved exactly across devices and OSes
- Looks professional and polished
- Resistant to unauthorized modifications
- "Most ATS read and parse PDF resumes more accurately" (Jobscan testing)

### PDF Disadvantages
- Hard to edit post-creation
- Larger file size
- No change-tracking

### DOCX Advantages
- Easily editable
- "Many ATS can effectively read information from Word resumes"
- Required by some job postings (Oracle Taleo, older iCIMS installs, some federal/government portals)

### DOCX Disadvantages
- Formatting can shift between Word versions and across OSes
- Jobscan notes: "some ATS systems had some issues with formatting and accurately parsing special characters when it comes to .docx files"
- Risk of unauthorized alterations after submission

### When to Use Which
- **PDF**: Default for every submission unless the posting says otherwise. Also use when emailing recruiters directly.
- **DOCX**: Only when the job posting explicitly requires `.docx`, or for internal HR tools (SuccessFactors, Taleo) that are known to strip PDF formatting.

### Rule for ApplyPilot's Pipeline
Generate both formats. Ship DOCX when the posting's upload field accepts it and the ATS is Workday, SuccessFactors, or Taleo (where older parsers prefer DOCX). Default to PDF otherwise.

Sources:
- https://www.jobscan.co/blog/resume-pdf-vs-word/
- https://www.jobscan.co/blog/20-ats-friendly-resume-templates/ (Jobscan's checklist also accepts "text-based PDF or .docx")

## 3. File Name

**Format:** `FirstName_LastName_JobTitle.pdf` or `FirstName-LastName-JobTitle.pdf`.

Source: https://www.jobscan.co/blog/writing-a-resume-pay-attention-to-the-file-name/

**Include:**
- First + last name
- Job title being applied for
- Keywords aligned with the job description

**Avoid:**
- Version numbers (`Resume_2023_v1.docx`)
- Generic names (`RESUME.docx`, `Final Resume.pdf`)
- Special characters (`*`, `|`)
- File names over ~24 characters
- Combining words without separators (`JohnSmithResume.pdf`)

Why: recruiters see the filename throughout the hiring process. It's a first impression.

## 4. Structural Rules

### Section Order (standard 5 sections)

1. **Contact Information**
2. **Professional Summary**
3. **Work Experience**
4. **Education**
5. **Skills**

Source: https://www.jobscan.co/blog/resume-sections/

**Career-level variations:**
- **Entry-level:** Education first (before Work Experience)
- **Mid-level / experienced:** Work Experience first, as above
- **Career changers:** Emphasize transferable skills and certifications near the top; consider a hybrid format

### Section Headers — Use Standard Names Only

ATS parsers look for specific keyword buckets. Non-standard headers cause your Work Experience to get categorized under Education, or your Professional Summary to disappear entirely.

**Use these exact headers:**
- Work Experience (or Professional Experience)
- Education
- Skills
- Professional Summary (or Summary)
- Certifications

**Never use creative headers like:**
- "Where I've Been"
- "The Toolkit"
- "My Journey"

Source: https://www.jobscan.co/blog/ats-formatting-mistakes/

### Contact Info Block
- **Full name** — largest font on the page, most prominent
- **City, state, zip** — full street address NOT required
- **Professional email** — `first.last@domain` style, no cutesy handles
- **Single phone number**
- **LinkedIn URL** (optional but recommended)
- **Portfolio / GitHub URL** (optional, role-dependent)

**CRITICAL:** Never put contact info in the document header or footer. ATS parsers skip those regions entirely. Put everything in the main body.

Source: https://www.jobscan.co/blog/ats-formatting-mistakes/

### Professional Summary
- **2-3 sentences max** (Jobscan's recommendation)
- Treat it as your elevator pitch
- Lead with your field + years of experience
- Include the target job title and top 2-3 hard skills verbatim from the posting
- Mention a quantifiable accomplishment

### Work Experience Bullets

Each bullet should contain:
1. **A strong action verb** (designed, initiated, executed, accelerated, led, built, shipped, reduced, increased)
2. **A keyword or skill** from the job description
3. **A measurable result** (number, percentage, dollar amount, time saved)

Source: https://www.jobscan.co/blog/ats-resume/

Target 2-3 measurable accomplishments per role.

### Education
- Degree, institution, location, year of completion
- **GPA only if 3.5 or higher** AND you're a recent graduate
- Honors/achievements optional

### Skills
- Include both hard and soft skills from the job description
- Target **5-10 total skills** for balance
- Match the posting's exact wording

## 5. Formatting Rules

### Fonts
**Approved fonts:** Arial, Calibri, Cambria, Garamond, Georgia, Helvetica, Tahoma, Times New Roman, Verdana.

Source: https://www.jobscan.co/blog/20-ats-friendly-resume-templates/

- **Body text:** 10-12 pt
- **Headings:** 14-16 pt
- **Minimum size:** 10 pt (never smaller)

Jobscan's resume formats guide also suggests sans-serif (Arial, Calibri) as the general default.

### Margins & Spacing
- **1 inch on all sides** (minimum 0.5 inches acceptable)
- Maintain white space for readability
- Single-line spacing inside bullets; blank line between sections

### Length
- **One or two pages are both ATS-compliant.**
- One page for early-career / <10 years experience
- Two pages for experienced professionals with multiple relevant roles
- No three-page resumes. Ever.

Source: https://www.jobscan.co/blog/20-ats-friendly-resume-templates/

### Bullet Point Symbols
**Use:** solid circles (●), open circles (○), squares (■), or dashes (—).

**Avoid:** custom symbols, checkmarks (✓), arrows (→), emoji, or decorative icons. They parse as gibberish or get dropped.

### Dates
**Use:** `Jan 2021 – Mar 2023` or `01/2021 – 03/2023`.

**Avoid:**
- `Jan '21 – Mar '23` (apostrophes confuse the parser)
- `2021 – 2023` (missing months → parser assumes Jan 1 to present with no year)
- `1/2021 – 3/2023` (single-digit months are borderline; prefer two-digit)

Source: https://www.jobscan.co/blog/ats-formatting-mistakes/

Consequence of bad date formatting per Jobscan: "The system leaves your Work History section completely blank, or it thinks you worked at your last job from January 1 to Present with no year attached."

## 6. What NOT to Include

These break ATS parsing. All are explicitly called out by Jobscan.

| Element | Why it breaks parsing |
|---|---|
| **Tables** | Parser reads across the whole page, mashing unrelated cells into one sentence |
| **Multi-column layouts** | Same issue — text reads out of order, scrambled like "a shuffled deck of cards" |
| **Headers / footers with content** | ATS skips these regions entirely |
| **Text boxes** | Many parsers ignore layered elements |
| **Graphics, images, logos** | Not readable as text data |
| **Skill bar graphs** (e.g., "80% Java") | Not parseable; use text "(Expert)" instead |
| **Icons / emoji** (📞 for phone) | Turn into `[NULL]` or random characters; "Profile" becomes "Pro?le" |
| **Custom / decorative fonts** | Non-web-safe fonts parse as garbled characters |
| **Photos of yourself** | Not parseable; also risks illegal bias signals in US |
| **Color-coded sections / infographics** | Not parseable |
| **Clever section titles** | See section 4 — use "Work Experience", not "My Journey" |

Sources:
- https://www.jobscan.co/blog/ats-formatting-mistakes/
- https://www.jobscan.co/blog/resume-tables-columns-ats/

### The Notepad Test
Jobscan's ATS-conversion rule of thumb:

> "If you can't highlight and paste the text into a basic Notepad file without it looking like a mess, the ATS can't read it either."

Source: https://www.jobscan.co/blog/convert-your-resume-to-an-ats-friendly-format/

After writing your resume, copy it all and paste into a plain text editor. If merged words, broken symbols, or missing content appear — the ATS sees the same thing.

## 7. Keyword Strategy

### Sources of Keywords
1. **Job title** — include it verbatim in your headline/summary
2. **Hard skills** — tools, technologies, certifications, methodologies
3. **Required qualifications** — degrees, years of experience
4. **Repeated terms** in the posting — if they say it twice, use it
5. **Synonyms + acronyms** — include both (`Search Engine Optimization (SEO)`)

### Where to Place Keywords

| Location | Purpose |
|---|---|
| **Resume headline / title line** | Match the target job title exactly — 10.6x interview multiplier |
| **Professional Summary** | Top 3-5 keywords in natural prose |
| **Skills Section** | All relevant hard skills, exact wording |
| **Work Experience Bullets** | Keywords embedded in accomplishment statements (not just listed) |

Source: https://www.jobscan.co/blog/top-resume-keywords-boost-resume/

### Keyword Stuffing — What It Looks Like and Why to Avoid

**Stuffing (bad):**
> "Experienced Python Python developer with Python skills using Python to build Python applications."

**Natural integration (good):**
> "Built a real-time fraud detection pipeline in Python, cutting false positives by 34%."

ATS don't always recognize synonyms. Write exact matches *when natural* — don't contort sentences.

### Hard Skills vs Soft Skills — How to Prioritize

Jobscan data: **76.4% of recruiters filter by skills. 88% of hiring managers prioritize hard skills specifically.**

Source: https://www.jobscan.co/blog/soft-skills-vs-hard-skills/

**Rule:** treat hard skills as the primary ATS optimization target. Soft skills matter but have less ATS weight.

### Soft Skills Section — Yes, But Demonstrate Them

Jobscan recommends a dedicated "Soft Skills Highlights" section with **3-5 accomplishment bullets**, NOT a bare list.

**Bad:** "Team player, communicator, organized."

**Good:** "Trained 450+ students in computer skills, coordinating with 6 teachers across 3 schools."

Source: https://www.jobscan.co/blog/soft-skills-resume/

### Skill Level Indicators

**Use:**
- Subheadings: "Expert", "Proficient", "Advanced Beginner"
- Parentheses inline: `Python (Expert), Rust (Proficient), Go (Beginner)`

**Never use:**
- Skill bars / progress bars / percentages (not ATS-parseable, visual-only)

Source: https://www.jobscan.co/blog/levels-of-skill-on-resume/

## 8. Resume Format Decision

Three formats, Jobscan's guidance:

| Format | Best for | ATS friendly? |
|---|---|---|
| **Chronological** | Experienced professionals with steady career progression | Yes — "preferred by recruiters" |
| **Functional** (skills-first, dates de-emphasized) | Career changers, gap explanations | **Risky** — "can have ATS-compatibility issues" |
| **Hybrid / Combination** | Career changers WITH relevant technical skills | Yes |

**Default recommendation:** **Chronological** unless you have a specific reason to use hybrid. Never use pure functional.

Source: https://www.jobscan.co/resume-formats

## 9. Multi-Role "Umbrella" Formatting

For multiple roles at the same company:

```
Acme Corp | San Francisco, CA | Jan 2019 – Present
  Senior Engineer | Jan 2022 – Present
    - Bullet...
    - Bullet...
  Engineer | Jan 2019 – Dec 2021
    - Bullet...
```

Parser attributes all roles to one employer. Do NOT use a two-column table to align roles side by side.

Source: https://www.jobscan.co/blog/resume-tables-columns-ats/

---

## 10. Skill Listing — Single-Column Safe Methods

When listing skills in a line (instead of bullets), use these separators — all parse cleanly:

- **Vertical bars:** `Python | Rust | Go`
- **Commas:** `Python, Rust, Go`
- **Middle dots:** `Python • Rust • Go`

Do NOT use two-column layouts to squeeze more skills on the page.

Source: https://www.jobscan.co/blog/resume-tables-columns-ats/

---

## 11. Final Resume Checklist

Before shipping, verify:

- [ ] Match rate is 75-85% for this specific posting (80% target)
- [ ] Job title appears verbatim somewhere in the top third of the resume
- [ ] Single column, no tables, no text boxes, no graphics
- [ ] Contact info in the main body (not the header/footer)
- [ ] Section headers are standard (Work Experience, Education, Skills, etc.)
- [ ] Dates in `Month YYYY – Month YYYY` format
- [ ] Fonts are web-safe (Arial, Calibri, etc.), body 10-12pt
- [ ] 1-inch margins
- [ ] File format is PDF (or DOCX if posting requires)
- [ ] Filename is `FirstName_LastName_JobTitle.pdf`
- [ ] One or two pages, never three
- [ ] Every bullet has a number, action verb, and job-description keyword
- [ ] Copied-and-pasted into Notepad, the text is coherent and complete
- [ ] Hard skills from the posting appear in summary, skills section, AND work experience
